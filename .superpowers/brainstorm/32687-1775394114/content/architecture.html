Архитектура данных и технические решения

Как устроено «под капотом» — ключевые решения, которые я принял за вас

1. Слой данных — CMS-ready без CMS

src/data/
  projects.ts   ← TypeScript объекты с реальными Unsplash фото
  materials.ts
  services.ts
  team.ts
  reviews.ts
src/types/index.ts  ← интерфейсы = будущие Sanity схемы 1:1
Принцип: компоненты импортируют из @/lib/content — абстракция, которая сейчас отдаёт из src/data/, а после — из Sanity. Ни один компонент не знает откуда данные.

2. Анимационная стратегия

Framer Motion
• Page transitions (opacity + y)
• Hover-эффекты на карточках
• Mobile menu drawer
• Dropdown submenus
• Stagger сетки проектов
• WhatsApp button appear
GSAP + ScrollTrigger
• Hero text reveal (SplitText)
• Ken Burns на hero фото
• Counter animation (УТП секция)
• Parallax на CTA фоне
• Philosophy text scrub
• Pinned sections (если уместно)
Lenis smooth scroll — обёртка всего приложения. prefers-reduced-motion отключает всё.

4. Формы и лидогенерация

3 точки захвата:

1
Страница /contacto — полная форма (Имя, Email, Телефон, Сообщение, Услуга)
2
«Хочу похожий проект» — мини-форма на страницах проектов (Имя, Тел, Email)
3
Modal CTA — «Solicitar presupuesto» из Hero и Footer (те же поля что форма 1)
React Hook Form + Zod → API Route → Resend SDK → email на info@conceptofino.com
Fallback если Resend не настроен: данные в console.log (не блокирует разработку)
